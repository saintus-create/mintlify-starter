## Usage Guidelines

### When to Use Filled Variants

Use filled variants (`.fill`) for:
- **Primary actions**: Main CTAs and important features
- **Navigation items**: Top-level navigation and anchors
- **Emphasis**: When you want to draw attention to specific content

### When to Use Outline Variants

Use outline variants (no `.fill`) for:
- **Secondary actions**: Less prominent features
- **Inline references**: Icons within text
- **Subtle indicators**: Supporting information

### Icon Selection Best Practices

1. **Semantic Clarity**: Choose icons that clearly represent the concept
2. **Consistency**: Use the same icon for the same concept throughout
3. **Visual Balance**: Mix filled and outline variants for hierarchy
4. **Context Awareness**: Consider the surrounding content and layout

## Adding New Icons

When adding new content, follow these steps:

<Steps>
<Step title="Choose the right icon">
  Browse [SF Symbols](https://developer.apple.com/sf-symbols/) to find an appropriate icon that matches your content's purpose.
</Step>

<Step title="Use consistent naming">
  SF Symbols use dot notation (e.g., `icon.variant.modifier`). Common modifiers:
  - `.fill` - Filled variant
  - `.circle` - Circular background
  - `.square` - Square background
</Step>

<Step title="Test in context">
  Preview your icon in both light and dark modes to ensure it looks good in all contexts.
</Step>

<Step title="Update this reference">
  Add your new icon to this reference page to maintain consistency across the team.
</Step>
</Steps>
